# Contributing Guide

First of all, thank you for considering contributing to this project! We welcome contributions of all kinds, including bug fixes, feature requests, documentation improvements, and code enhancements.

## Before You Start

Please ensure that you:

* Read the project documentation.
* Search existing issues before creating a new one.
* Follow the project's coding standards.
* Test your changes before submitting them.

## Reporting Bugs

When reporting a bug, please include:

* A clear and descriptive title.
* Steps to reproduce the issue.
* Expected behavior.
* Actual behavior.
* Screenshots or logs (if applicable).
* Environment details (Operating System, Browser, Python version, etc.).

## Suggesting Features

Feature requests are always welcome.

Please provide:

* A detailed description of the feature.
* The problem it solves.
* Possible implementation ideas (optional).
* Any alternatives you considered.

## Development Setup

1. Fork the repository.
2. Clone your fork.
3. Create a new branch.

```bash
git checkout -b feature/your-feature
```

4. Install dependencies.

```bash
pip install -r requirements.txt
```

5. Configure your environment variables.

6. Run the development server.

7. Make your changes.

8. Run all tests.

9. Commit your work using clear commit messages.

Example:

```text
feat: add wallet address validation

fix: resolve login redirect issue

docs: update installation guide
```

## Pull Requests

Before submitting a Pull Request, ensure:

* Your code follows the project's style.
* All tests pass.
* Documentation has been updated if necessary.
* Your branch is up to date with the latest main branch.

Every Pull Request should include:

* A summary of the changes.
* Screenshots (if UI changes were made).
* Related issue number (if applicable).

## Coding Standards

* Write clean and readable code.
* Keep functions small and focused.
* Use meaningful variable names.
* Add comments where necessary.
* Remove unused imports and files.

## Documentation

If your contribution changes functionality, update the relevant documentation.

## Questions

If you have any questions, open a discussion or create an issue.

Thank you for helping make this project better!
